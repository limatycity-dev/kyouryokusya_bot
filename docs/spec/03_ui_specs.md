# 03_ui_spec — 文明BOT UI仕様（全体方針）

文明BOT の UI は  
**「文明の活動を美しく、わかりやすく可視化する」**  
ことを目的として設計されている。

本章では、Embed・ボタン・モーダルなどの  
**文明全体に共通する UI 原則（憲法）** を定義する。

個別 Feature の UI（クエスト、ランキング、面接など）は  
03_features の各章に記載する。

---

# 1. UI の基本思想（UI Philosophy）

文明BOT の UI は以下の価値観を最優先とする：

1. **文明の状態を美しく可視化すること**  
2. **情報を整理し、迷わない UI を提供すること**  
3. **文化として統一されたデザインを持つこと**  
4. **Embed を中心とした視認性の高い UI を構築すること**

---

# 2. Embed 使用基準（必須ルール）

文明BOT では、以下の場合 **必ず Embed を使用する**：

- 文明の状態を表す情報  
- クエスト（作成・編集・完了・終了）  
- ランキング（総合・週間）  
- setup 完了通知  
- 重要イベント通知  
- エラーメッセージ（赤）

### テキストで十分なケース
- 軽い通知  
- register の結果  
- admin の軽い設定変更  
- 内部的な成功メッセージ  

---

# 3. 色のルール（Color Guidelines）

文明BOT の UI は **ベージュ系の文化色** を基調とする。

| 用途 | 色 |
|------|------|
| 成功 | 青 |
| エラー | 赤 |
| 文明の状態 | 青 or 灰 |
| 警告 | 黄色 |
| ランキング・クエスト | ベージュ系 |

---

# 4. ボタンの文化的ルール

ボタン色は文明内で統一された意味を持つ。

| 色 | 意味 |
|------|------|
| 緑（Success） | 完了・承認 |
| 赤（Danger） | 終了・削除 |
| 青（Primary） | 主要操作 |
| グレー（Secondary） | 更新・補助操作 |

---

# 5. モーダルの原則

- 入力項目は最小限  
- 必須項目と任意項目を明確にする  
- 入力値はバリデーションする  
- モーダルは「操作の補助」であり、状態表示は Embed に任せる  

---

# 6. UI と DB / State の同期原則

- Embed は **文明の状態の唯一の可視化手段**  
- DB の状態と Embed の表示は常に一致させる  
- message_id により、Embed を安定して更新する  
- スレッド名・ボタン状態も DB と同期する  

---

# 7. UI の責務分離

- **UI の詳細仕様は Feature 側（03_features）に記載する**  
- 05_ui_spec は文明全体の UI 原則のみを扱う  
- Feature は UI / DB / Commands / Flow を 1 つの棚として完結させる  

---

# 8. 今後の拡張（任意）

- 文明全体のカラーガイドライン策定  
- UI コンポーネントの共通テンプレート化  
- 月間ランキング UI の追加  
- クエスト一覧 UI の追加
